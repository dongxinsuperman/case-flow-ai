# 助手对话模式

## 定位

助手把自然语言请求映射到 Case Flow 已注册能力。它不是公司内部运维机器人，也不包含任何私有账号、CLI 或基础设施知识。

## 官方工具

- AI Phone 执行与设备查询。
- AI Web 执行与浏览器槽查询。
- AI API 执行。
- 飞书缺陷提交。

CLI 只是一种可选 Tool，不是助手主功能。开源版本不内置公司的 CLI Tool。测试账号、权益或业务数据准备请求不会被误派给手机、网页或接口执行器，而是明确说明该能力需要部署方二次开发；未接入不影响助手使用其他工具。

## 决策和执行边界

1. 工具列表由后端运行时生成，模型不能调用未注册工具。
2. 参数不足时明确追问；能力不存在时明确说明需要配置或二次开发。
3. 工具调用失败必须作为失败结果返回，不能换成规则假成功。
4. Function Map 只在工具确定后按消息引用解析并注入，不作为模型臆测其它业务能力的依据。
5. 飞书、豆包、AI Phone、AI Web 是当前官方维护基线；其他平台和模型由使用方自行开发。

## 数据隔离

助手消息、上传图片、执行记录和 bug 记录使用独立 `agent_*` 数据表。上传物和报告属于运行数据，必须 Git 忽略。公开仓库只包含协议、业务编排和无真实数据的测试。

## 缺失能力表现

- 豆包未配置：不能进行模型工具决策；只保留明确关键字可判断的公共工具路由，模型生成类能力不可用。
- 业务数据工具缺失：显示“当前开源版本未提供，需要部署方二次开发”。
- AI Phone / AI Web 不可达：显示设备或提交接口的真实错误。
- 飞书未配置：禁止提交缺陷并列出缺失配置。
