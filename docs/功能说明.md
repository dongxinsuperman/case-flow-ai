# 功能说明

本文只描述开源版本当前能够使用的功能和依赖边界。

## 两条使用路径

### 快速模式

快速模式用于把一份 Markdown 用例直接导入独立工作区，完成结构化查看、规则打标、编辑整理和导出；配置相应能力后，还可以继续执行和诊断。它不要求先建立飞书项目、一级目录或二级需求，可以在没有飞书配置时作为独立的 Case 打磨工作台使用。

基础能力只依赖：

- PostgreSQL。
- 豆包/Ark 模型不是导入的强依赖：明确标签和关键字规则优先；未配置模型时，规则无法识别的 Case 归为人工并继续导入。

飞书不是快速模式的启动依赖。未配置飞书时，不能使用项目池、需求目录同步、需求绑定和飞书 Bug 闭环，但不影响以下 Case 打磨能力：导入、规则打标、查看、编辑和导出。

其他可选能力：

- CLI 只是一种可选 Tool。开源版本不内置公司的 CLI Tool；依赖企业业务数据工具的任务会明确要求二次开发，其余能力正常。
- 不部署某个外部执行器：对应的执行按钮会得到真实不可用错误，不影响 Markdown 管理能力。

详见 [快速模式](快速模式.md)。

### 标准工作流

标准工作流围绕“项目来源 → 需求目录 → Case 资产 → 执行 → 报告 → 修复或 Bug”组织。目前只维护飞书项目来源和飞书缺陷提交。其他项目系统需要使用方自行开发。

## 当前能力矩阵

| 能力 | 数据库 | 模型 | 飞书 | 外部执行器 |
|---|---:|---:|---:|---:|
| 健康检查、配置状态 | 必需 | 可选 | 可选 | 可选 |
| 快速模式 Markdown 导入/编辑/导出 | 必需 | 可选 | 不需要 | 不需要 |
| 快速模式执行与诊断 | 必需 | 诊断需要 | 不需要 | 需要所选执行器 |
| 快速模式提交飞书 Bug | 必需 | 需要 | 需要 | 需要已有失败证据 |
| 标准项目/需求同步 | 必需 | 可选 | 需要 | 不需要 |
| AI Hybrid | 必需 | 需要 | 不需要 | 取决于任务调用的工具 |
| 助手对话 | 必需 | 需要 | 仅飞书操作需要 | 取决于任务调用的工具 |

“可选”不表示静默跳过。用户选择了依赖该能力的操作时，系统必须明确返回未配置、不可用或需要二次开发。

## 主要功能

- Markdown Case 导入、结构化编辑和导出。
- 标准模式 Markdown 层级配置，以及快速模式动态路径层级解析。
- 重复导入时的碰撞审批与当前工作集维护：完全一致的旧 Case 会先全局锁定；其余差异再进入 1:1 替代、新增、删除或保留审批，避免相似 Case 抢占完全一致的旧 Case。
- 碰撞展示与确认入库使用同一份审批快照。文件内容、来源或旧 Case 批次在审批期间变化，或快照已过期时，系统明确要求重新导入碰撞；不会重新猜测、静默重算或按过期结果写入。
- Case 脑图、执行状态、覆盖标记和执行流水。
- Function Map 资产管理、挂载和执行时上下文编译；AI Hybrid 可按已挂载的账号/角色映射精确锁定 AI Phone 设备，绑定不明确或设备不可用时明确转人工，不换设备猜测。
- AI Phone、AI Web、AI API、AI Hybrid 执行。
- 失败证据读取、模型诊断和可审批修复草稿。
- 飞书项目需求同步与飞书缺陷提交。
- 豆包/Ark 模型调用。

## 不提供的能力

- 公司账号、SSO、内部 CLI、内部域名和私有执行器实现。
- 钉钉、Jira、禅道等项目系统的适配与支持。
- 除豆包/Ark 外模型的兼容承诺。
- 缺少能力时的假成功、默认猜测或静默降级。
