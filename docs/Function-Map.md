# Function Map

Function Map 是提供给 AI 执行器的可复用业务上下文卡片，例如页面规则、接口约定、登录方式、账号使用约定和测试注意事项。

它解决的问题不是“让模型知道所有东西”，而是让团队明确控制：什么业务知识可以复用、挂在哪个测试范围、执行时应该交给哪个端，以及出问题后如何核对。

## 当前使用流程

1. **建立卡片**：在 Function Map 资产库填写标题、适用场景、正文和适用端。
2. **显式挂载**：把卡片挂到一级目录、二级需求或快速模式 Session。
3. **执行时编译**：系统根据本次执行范围和执行端筛选、去重并合成上下文。
4. **流水核对**：在执行流水中查看本次实际提交的 Function Map；为空时可以继续判断是没有挂载、适用端不匹配，还是提交前失败。

同一张卡片可以被多处引用。修改资产后，所有引用在后续执行时都会读取新内容，不需要重复上传文件。

正文可以直接填写，也可以导入本地 `.md`、`.txt` 或 `.json` 文本；导入文件会覆盖编辑区正文。每次保存都会记录最近一次写入方式：直接填写显示为 `manual`，本地文件导入显示为 `local_import` 并保留文件名。两种方式都只改正文，不会意外修改标题、适用场景或适用端。

## 一张卡片包含什么

| 字段 | 作用 |
|---|---|
| 标题 | 用于识别和检索卡片 |
| 适用场景 | 说明什么任务应该使用这份上下文 |
| 适用端 | `App`、`Web`、`API`，可多选 |
| 正文 | 实际交给执行器的业务说明 |

列表只读取摘要，正文按需查看或导出，避免在资产列表中反复搬运完整内容。

## 挂载范围

### 标准模式

- 一级目录挂载：其下二级需求执行时可以继承。
- 二级需求挂载：只对该需求生效。
- 同一资产同时出现在一级目录和二级需求时只带一次，来源记为一级目录继承。

### 快速模式

快速会话直接从资产库选择需要的卡片。快速会话没有一级、二级目录继承关系，执行时只编译当前会话显式选择的资产。

## 按执行端编译

| 执行器 | 使用的适用端 |
|---|---|
| AI Phone | `App` |
| AI Web | `Web` |
| AI API | `API` |
| AI Hybrid | `App`、`Web`、`API` |

编译时会按资产 ID 去重，给每份正文加上标题、资产 ID 和来源边界，再合成为本次提交的 `functionMapContext`。AI Hybrid 同时接收同一批资产的结构化数据；模型首轮只看到目录（ID、标题、简介、适用端），再通过工具逐份读取全部正文。每份 Map 只作为对应 App、Web 或 API 端的编排参考：它帮助 Hybrid 理解任务和组织子任务，但不是固定脚本，也不能替 Hybrid 决定流程、调用顺序或最终结论。

当前实现只读取**显式挂载**，不做模型自动发现。没有匹配资产时不携带 Function Map，不读取隐藏全局文件、不猜测业务规则，也不静默截断上下文。

如果 Map 指定了某个账号或角色必须使用的设备，AI Hybrid 会把这次 AI Phone 子任务硬锁到该 alias。设备不可用、平台不一致、别名不唯一或限时等待后仍占用时会明确停止并要求人工处理；不会把任务改派到另一台设备。未指定绑定的 Case 仍按普通设备池运行。

## 执行流水为什么重要

每次执行都会留下调用台账，可以核对：

- 本次使用标准模式还是快速模式。
- 交给了哪个执行器。
- 关联了哪个需求或快速会话。
- 实际提交了哪些 Function Map 内容。
- 编译或提交失败发生在哪一阶段。

这样可以区分“没有挂载”“端不匹配”“Case Flow 没有提交”“执行器没有消费”等不同问题，避免用执行结果反推上下文是否生效。

## 简单例子

团队把“登录测试账号与验证码规则”建成一张卡，适用端选择 App 和 Web，再挂到“账号登录”需求。执行 App Case 时，AI Phone 收到这张卡；执行 Web Case 时，AI Web 也收到；执行 API Case 时，因为适用端不匹配而不会携带。执行完成后，可以在流水中核对本次到底提交了什么。

## 当前不做什么

- 不让模型在整个资产库中自动猜测应该使用哪张卡。
- 不使用隐藏的全局 Function Map 兜底。
- 不因为 Function Map 缺失而假装执行器已经理解业务。
- 不把 Function Map 当作通用 Agent Skills；它服务于垂类测试执行上下文。

自动发现、离线推荐和执行器运行时懒加载属于未来方向，当前不能当作已实现能力使用。

## 数据边界

运行时创建和挂载的资产保存在数据库，不属于公开仓库内容。部署方应自行管理其中的账号、业务说明和测试数据，不能把本地真实资产导出后提交到公开 Git。

接口清单见 [API 契约](API契约.md)，当前提交字段和 Hybrid 透传规则见 [Function Map 执行上下文协议](function%20map%20skills化执行单元协议.md)。
