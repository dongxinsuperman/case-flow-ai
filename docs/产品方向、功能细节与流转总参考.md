# Case Flow 产品方向、功能细节与流转总参考

这份文档是个人项目 `case-flow` 的**产品行为**权威参考。正式工作流按三个 Tab 组织：项目页、Case 页、首页；快速模式作为独立路由单列，不纳入正式需求治理链路。

范围说明：本文件只讲**产品行为**（功能点、流转、规则、边界）。**技术与契约细节**（数据模型、API 契约、AI Phone 集成、开发与运维）不写在这里，见 `docs/索引.md`（文档总导航）与 `../AGENTS.md`（项目入口）。新增/修改产品行为时同步更新本文件，并用状态标签标注。

写法约定：正式工作流 Tab 是第一层级，Tab 内功能块是第二层级，具体功能点是第三层级；快速模式按独立路由同级记录。功能点下面的状态、目标、用户操作、系统表现、数据变化、规则、边界、待定项都缩进到该功能点下面，不和功能点平级。

细节标注约定：功能点下面的每条细节都要用状态标签开头，例如 `[当前]`、`[规划]`、`[待定]`、`[需修正]`。状态标签后面可以继续跟事项标签，例如 `[当前][用户操作]`、`[规划][解析]`、`[待定][方案]`。同一个功能点可以同时包含当前能力、规划能力和待定方案。

功能点状态说明：`当前` 表示已经实现；`需修正` 表示当前实现和期望不一致；`规划` 表示后续明确要做；`待定` 表示方向存在但规则未拍板。

## 0. 快速模式：Markdown 快速打磨

路径：`/quick`

当前定位：独立于正式需求治理体系的 Markdown case 快速打磨页面。

### 0.1 快速模式入口与页面边界

- 功能点：首页进入快速模式
  - 状态：当前
  - [当前][目标] 给用户一个非常明显的快速打磨入口，用于处理“只想导入 Markdown、打磨 case、再导出 Markdown”的场景。
  - [当前][用户操作] 用户从首页点击快速模式入口，系统直接切换到独立快速模式路由。
  - [当前][系统表现] 快速模式页面不展示项目池、一级目录、二级需求、任务切换、functionMap、bug 等正式工作流功能。
  - [当前][边界] 快速模式不是首页的简化版本，也不是隐藏的未归属二级需求。
  - [当前][公开参考] 使用方式和依赖边界见 [快速模式](快速模式.md)。

- 功能点：Markdown 进出闭环
  - 状态：当前
  - [当前][目标] 形成“导入 Markdown -> 脑图打磨 -> 导出 Markdown”的闭环。
  - [当前][用户操作] 用户进入快速模式后，点击上传或拖入 Markdown。
  - [当前][用户操作] 用户可以在导入后的工作台给本次 session 选 Function Map：从资产库按引用挑选（带模糊搜索），存为 `function_map_quick_mounts` 挂载，不再临时上传文件；要新增/改正文都去 Function Map 资产库。旧的多文件上传（`function_files`）已停用。
  - [当前][用户操作] 用户可以填写 session ID 接力恢复未导出的 session。
  - [当前][系统表现] 导入前页面主体是大面积 Markdown 上传/拖入区；底部只保留 session 接力入口。
  - [当前][系统表现] 导入后页面主体切换为类似首页脑图的 case 结构视图，并展示执行总览卡片。
  - [当前][系统表现] 导入后页面展示当前 session ID，供复制接力。
  - [当前][系统表现] 页面顶部保留“导出 Markdown”和“退出”，工作台工具条提供飞书需求链接、Function Map 资产选择器（浮层，带模糊搜索）；执行、提交 bug、修复入口走 quick 专属能力。
  - [当前][系统表现] 飞书需求链接保存时立即解析校验；只有链接所属空间已在标准版完整配置需求和 bug 模板时，页面才提示可提交 bug。
  - [当前][系统表现] 未导出的 session 通过浏览器本地缓存，刷新页面不丢失。
  - [当前][数据规则] 快速模式数据不默认进入正式 case 资产池。
  - [当前][数据规则] 一个 session 只对应一份 Markdown，不支持重新导入、不做碰撞审批。
  - [当前][数据规则] 快速模式 Markdown 层级规则自成一套，不读取部署级 Markdown 层级配置，也不做 session 配置快照。
  - [当前][数据规则] 快速模式按位置识别最后四级：标题、前置条件、操作步骤、预期结果；最后四级之前的所有层级作为路径节点原样保留。
  - [当前][标题规则] 标题开头的 `【标签】` 可额外识别为结构化标签，但识别是非破坏性的；脑图、列表、编辑、碰撞、执行、诊断、Bug 和 Markdown 导出始终保留完整标题正文。
  - [当前][解析规则] 快速模式导入时，测试标题、前置条件、操作步骤、预期结果按单行字段保存；字段里的真实换行或非列表续行会折叠为空格。
  - [当前][执行端分类] 快速模式导入时复用标准导入打标：先按显式标题标签和内容关键词规则识别执行端；Web 内容规则包含平台、后台、管理端、控制台、站点/网站、cb、vm、视频后台、课程后台等业务词，不包含单独的“课程”。规则未命中的 case 同步进入模型打标，最多 3 个模型请求并发均分，只允许输出 `app/web/api/manual`，不含 AI Hybrid。模型只接收 id、标题、前置条件，单请求超时 90 秒；模型不可用、超时、解析失败或返回非法枚举时按人工执行兜底。
  - [当前][数据规则] quick session 后端持久化到 quick 专属表，前端本地缓存 session ID 用于自动恢复和接力。
  - [当前][数据规则] session ID 是接力凭证，必须是高熵随机值，不能用自增 ID 或可猜测短码。
  - [当前][数据规则] 提交 bug 前必须已选择提交人，并且飞书需求链接已保存、可解析、命中标准版完整配置；缺少或解析失败时只弹前置信息，不进入提交 bug 表单。
  - [当前][编辑规则] 导入后只允许编辑测试标题、前置条件、操作步骤、预期结果；测试集标题和路径层级只展示，不允许修改；不允许新增、删除或拖拽排序 case。
  - [当前][编辑规则] 编辑保存和修复候选应用时，测试标题、前置条件、操作步骤、预期结果里的真实换行折叠为空格。
  - [当前][编辑规则] 编辑保存不重新判定执行端，保留导入打标结果或用户手动选择的执行端。
  - [当前][编辑规则] 编辑保存和修复候选应用后，执行状态重置为未执行，旧报告、覆盖、失败分析、修复草稿、bug 草稿、执行关联、执行时间和旧执行 item 可查结果被清空。
  - [当前][导出规则] 导出的是导入 Markdown 被编辑后的内容，按快速模式自己的路径层 + 末尾四段结构导出，不导出 function、飞书链接、操作者、session 信息、执行结果或 bug 信息。
  - [当前][导出规则] 导出的每个 Markdown 节点都是单行列表项，字段内容不输出真实换行，确保快速模式导出后可以再次导入。
  - [当前][清理规则] 用户成功导出 Markdown 或确认退出快速模式时清空 session，导出成功后不二次确认，退出需要确认。
  - [当前][执行规则] AI Phone / AI Web / AI API 执行不要求飞书需求链接，也不要求选择操作者；导出不因执行状态阻断，孤儿回调需被容忍。
  - [当前][执行规则] quick 的 AI API case 会提交到 `/api/v1/quick/executions/aiapi/submit`，由内置 AI API 后台执行并回写 quick 专属执行状态，不再只记录本地 running。
  - [当前][bug规则] 飞书需求链接不是导入必填项；用户触发提交 bug 前才要求填写、保存并解析。提交 bug 时必须选择提交人；需求链接无效、空间未接入标准版完整配置或 bug 模板缺失时阻断提交。
  - [当前][边界] 执行器、bug、报告修复等能力必须围绕 quick session 单独实现，不复用标准执行、bug、修复服务编排。

## 0A. OS Agent 对话模式：全局轻量能力入口

路径：全局浮窗，挂在 `App.vue` 级别；标准模式跟随右上角当前用户，快速模式跟随 quick session 当前用户；没有当前用户的页面隐藏。

当前定位：独立于正式工作流和快速模式的单会话能力调用助手。

### 0A.1 单会话入口与消息体验

- 功能点：当前用户绑定单会话
  - 状态：当前
  - [当前][目标] 用户随手输入一句自然语言，由助手判断是否调用工具并把结果回到同一个对话里。
  - [当前][配置] OS Agent 受 `CASE_FLOW_OS_AGENT_ENABLED` 总控；关闭时前端不展示入口，后端 agent 接口不可用。
  - [当前][用户操作] 用户点击右下角 `OS Agent` 浮动按钮打开面板。
  - [当前][用户操作] 用户点击面板外空白或其他页面区域时，面板自动收回。
  - [当前][系统表现] 一个用户只有一个助手会话；助手跟随右上角当前操作者，不在助手面板里再选用户。
  - [当前][系统表现] 如果页面没有可见的当前操作者，助手不展示。
  - [当前][系统表现] 助手未打开时，新的非用户消息通过红点计数提示；点开即视为已读。
  - [当前][用户操作] 用户可在面板内点击“清空上下文”清空当前人的 OS Agent 上下文。
  - [当前][系统表现] 清空上下文会删除当前会话消息、工具调用记录、自然语言 bug 快照和临时状态，并写入一条新的能力引导消息。
  - [当前][数据规则] 消息、工具调用、bug 提交快照全部写 `agent_*` 表，不写正式 `case_*`、`aiphone_execution_*` 或 quick 表。

- 功能点：图片作为对话附件
  - 状态：当前
  - [当前][用户操作] 用户可以选择、粘贴或拖拽图片到助手输入框。
  - [当前][系统表现] 图片在待发送区和消息气泡内以缩略图展示，可点击打开原图。
  - [当前][数据规则] 图片落在 `/media/agent_uploads/`，消息只保存 URL、文件名、mime、size 等附件元信息。

### 0A.2 工具调用与结果回复

- 功能点：助手工具范围
  - 状态：当前
  - [当前][能力] 当前注册工具包括 AI Phone、AI Web、AI API、资源查询和自然语言提 bug。
  - [当前][边界] CLI 只是一种可选 Tool。开源版本不内置公司的 CLI Tool 或企业业务数据工具；相关请求明确提示需要部署方二次开发，不影响其他工具。
  - [当前][能力] `kubectl` 只作为后续候选，不进入当前 MVP。
  - [当前][规则] 没有 LLM 工具调用能力时，后端使用保守关键词兜底；普通聊天不会默认提交执行器。
  - [当前][边界] 只开放已接入工具能力；Case Flow 平台自助操作器式内部功能调用不进当前范围。

- 功能点：执行结果回到对话
  - 状态：当前
  - [当前][系统表现] AI Phone / AI Web 回调或 AI API 后台完成后，助手写入一条自然语言 result 消息。
  - [当前][系统表现] 报告链接、关键截图和工具日志只是附件或证据，不直接替代助手回复。
  - [当前][边界] 一次只处理用户明确说的任务；不做“看完上一个工具结果后自动继续下一步”的自由 agent loop。

- 功能点：当前需求 functionMap 携带
  - 状态：当前
  - [当前][用户操作] 助手输入区提供“使用当前需求 function map”勾选项；标准首页存在当前二级需求时可用，quick 页面存在 quick session 时携带该会话选中的 Function Map。
  - [当前][系统表现] 前端每条消息只传当前工作台引用（标准模式 `requirement_item_id`，quick 模式 `quick_session_id`）和勾选状态，不传 functionMap 正文。
  - [当前][系统表现] OS Agent 模型决策阶段看不到 functionMap 正文；模型只判断要不要调用运行时已经注册的 AI Phone、AI Web、AI API 或 bug 工具。
  - [当前][系统表现] 工具决策后，后端按消息提交时携带的引用实时读取 functionMap，并在调用 AI Phone / AI Web / AI API 时注入；已提交任务锁定当时注入的文本，后续切换需求或修改 functionMap 不影响已提交任务。
  - [当前][规则] 标准模式读取「当前二级需求 + 其所属一级目录」的显式挂载资产编译（`compile_top_level_context`，去重、带资产边界）；quick 模式读取快速会话从资产库选中的资产编译（`compile_quick_context`，读 `function_map_quick_mounts`）；没有挂载、勾选关闭或引用失效时不注入，也不回落全局 env。旧 `quick_sessions.function_files` 已停用不再读。

### 0A.3 自然语言提 bug

- 功能点：助手自然语言提交 bug
  - 状态：当前
  - [当前][用户操作] 用户明确说“提 bug / 建单 / 发出去”时，助手进入提 bug 流程。
  - [当前][系统表现] 不弹强确认表单，不复刻手动提 bug 的字段编辑弹窗；标题、描述、字段由助手按对话、图片、报告和需求绑定尽力生成。
  - [当前][系统表现] 自然语言提 bug 的描述字段只承载缺陷证据；标题写入标题字段，飞书需求链接写入关联需求字段，不在描述里重复展示。
  - [当前][规则] 缺飞书需求链接、当前用户缺飞书 user_key、空间未配置 bug 模板等硬前置时，助手在对话里追问或阻断。
  - [当前][规则] 用户明确表达提 bug 且当轮带图但缺飞书需求链接时，助手只临时保存这一轮描述和图片，下一轮补需求链接则直接提交并带上这些图片；若下一轮转向其他任务或普通聊天，则丢弃该临时状态。
  - [当前][规则] 同一轮同时包含提 bug 意图、飞书需求链接和图片时，助手直接提交，不只做需求绑定。
  - [当前][边界] OS Agent 不做已提交 bug 的后续图片增删改；提交成功或清空上下文后，临时提 bug 状态结束。
  - [当前][边界] 自然语言提 bug 允许不如手动提 bug 精准；成功后只回 bug 链接和提交说明。

## 1. 项目页：飞书项目 / 目录管理

路径：`/requirements`

当前定位：外部项目池进入 Case Flow 目录体系的页面。

### 1.1 项目池

- 功能点：查看项目池
  - 状态：当前
  - [当前][目标] 展示当前可处理的外部项目条目。
  - [当前][系统表现] 项目页左侧展示项目列表。
  - [当前][系统表现] 项目池列表由后端按空间、测试人员、迭代、生命周期和页码筛选后返回；页面不一次性拉取全部项目池数据。
  - [当前][系统表现] 每个项目展示标题、描述和当前状态。
  - [当前][数据表现] 项目状态分为未纳入和已纳入。
  - [当前][数据表现] 未纳入表示项目还没有进入 Case Flow 的目录体系。
  - [当前][数据表现] 已纳入表示项目已经被挂到某个一级目录下，并形成对应二级需求。

- 功能点：区分待处理和已纳入项目
  - 状态：当前
  - [当前][目标] 避免同一个外部项目被重复纳入多个目录。
  - [当前][系统表现] 未纳入项目可以勾选。
  - [当前][系统表现] 已纳入项目不能再勾选。
  - [当前][系统表现] 已纳入项目展示它已经纳入到哪个一级目录。
  - [当前][规则] 已纳入项目不参与本轮创建目录或纳入已有目录操作。

- 功能点：多选待处理项目
  - 状态：当前
  - [当前][目标] 支持一次处理多个未纳入项目。
  - [当前][用户操作] 用户勾选一个或多个未纳入项目。
  - [当前][系统表现] 后续创建目录或纳入目录时，同时处理本轮勾选的项目。
  - [当前][边界] 已纳入项目勾选框禁用。

- 功能点：待处理数量提示
  - 状态：当前
  - [当前][目标] 让用户快速知道还有多少项目未纳入目录体系。
  - [当前][系统表现] 项目池区域显示待处理项目数量。
  - [当前][数据规则] 待处理数量只统计未纳入项目。

### 1.1A 未归属二级需求

- 功能点：飞书项目作为未进入目录的二级需求
  - 状态：当前
  - [当前][目标] 降低飞书项目进入 Case Flow 的前置门槛，让用户可以先导入和执行 case，后续再补一级目录治理。
  - [当前][业务口径] 飞书项目等于二级需求；`requirement_pool` 只是同步缓存，用户不需要感知技术补齐动作。
  - [当前][用户操作] 飞书项目列表每条提供「飞书」和「去 Case」入口；「去 Case」直达 Case 资产页并定位该二级需求。
  - [当前][系统表现] 未绑定一级目录的二级需求展示在“未进入目录”分区，并可以在 Case 页导入 Markdown。
  - [当前][数据规则] case 仍然必须归属二级需求；未进入目录只表示二级需求没有绑定一级目录。
  - [当前][数据规则] 飞书拉取时会同步确保每个入池项目有 1:1 `requirement_item`，默认 `group_id = null`；未上线阶段不做旧数据兼容承诺，开发期旧脏数据直接删库重拉。
  - [当前][数据规则] 项目池展示状态从 `requirement_item.group_id` 实时计算，不额外存一份“未归属/已归属”状态。
  - [当前][数据规则] 未进入目录二级需求不绕过首页任务过滤；只有已挂当前用户负责人且生命周期命中“测试中”的需求才进入首页。
  - [当前][数据规则] 未进入目录二级需求不要求 version，不继承一级目录 functionMap。
