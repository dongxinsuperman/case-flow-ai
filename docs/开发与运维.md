# 开发与运维

本地起服务、端口、环境变量、迁移、测试。

## 端口一览

| 服务 | 端口 | 属于 | 备注 |
|---|---|---|---|
| case-flow 前端（Vite） | 5173 | 本项目 | `/api` 代理到 8800 |
| case-flow 后端（FastAPI/uvicorn） | 8800 | 本项目 | 健康检查 `/api/v1/healthz` |
| AI Phone 后端 | 8000 | 外部依赖 | 由 AI Phone 项目启动，本仓库不含 |
| AI Web 后端 | 8009 | 外部依赖 | 由 AI Web 项目启动，本仓库不含 |

外部执行器是否启动不影响 Case Flow 后端和快速模式 Markdown 管理能力；只有选择相应执行器时才要求它可达。

## 后端

环境（不入库）：`backend/.venv`。推荐 Python 3.11。

```bash
# 安装（首次）
docker compose -f docker/docker-compose.yml up -d postgres
python3.11 -m venv backend/.venv
backend/.venv/bin/python -m pip install -e "backend[dev]"

# 配置
cp backend/.env.example backend/.env   # 按需改

# 迁移
cd backend && .venv/bin/alembic upgrade head

# 启动（带 --reload，改代码自动重载）
.venv/bin/uvicorn app.main:app --reload --host 127.0.0.1 --port 8800

# 测试
.venv/bin/pytest -q
```

数据库：PostgreSQL，默认连接见 `.env` 的 `CASE_FLOW_DATABASE_URL`。如使用本仓库 `docker/docker-compose.yml`，默认地址是 `127.0.0.1:55432`，库名和用户都是 `case_flow`。

只使用快速模式导入、编辑和导出时，最小必需配置只有数据库。豆包、飞书、AI Phone 和 AI Web 都是按操作启用的能力，详见 [快速模式](快速模式.md)。

## 前端

```bash
cd web
npm install        # 首次
npm run dev        # 开发（5173，HMR）
npm run build      # 构建校验（vue-tsc + vite，提交前应为绿）
npm run typecheck  # 仅类型检查
```

与另一个本地 Case Flow 实例并行运行时，可临时指定前端端口和代理目标，不需要修改 `.env`：

```bash
CASE_FLOW_DEV_BACKEND_URL=http://127.0.0.1:8810 npm run dev -- --host 127.0.0.1 --port 5174 --strictPort
```

> CSS/样式偶尔 HMR 残留旧值，看不到变化时 **Cmd+Shift+R 硬刷新**。

### 导入碰撞的运行约束

同文件二次导入的后台任务和审批快照当前都在后端进程内存中。因此部署必须保持**单 worker、单 Pod**；服务重启、快照过期或多实例错进程时，用户会收到“重新导入碰撞”的明确提示。不要用重算或自动提交掩盖该状态。需要横向扩容时，必须先将任务和快照迁移到共享存储。

### 执行取消的运行约束

AI Hybrid 和标准/快速模式 AI API 的取消句柄也只保存在后端进程内存中。服务重启、多 worker 或跨 Pod 调用时，系统不能确认原协程是否还存在，也不会把这种未知情况伪装成“已停止”。目前应保持单 worker、单 Pod；需要横向扩容或可靠跨进程取消时，必须将取消信号和运行句柄迁移到共享任务系统，并为外部执行器实现各自的撤销协议。

## 迁移规范

- 改表必须加 Alembic revision（`backend/alembic/versions/`），串好 `down_revision`，`alembic upgrade head` 应用，并更新 `数据模型.md`。
- 迁移以 `backend/alembic/versions/` 和 Alembic 当前 head 为准；不要在文档中手工维护可能过期的完整编号列表。

## 数据初始化（真实飞书数据）

不再有 mock seed 脚本。用户与负责人均由飞书拉取驱动：

1. 配 `backend/.env` 飞书凭证 + `backend/config/feishu_project.json` 空间/映射。
2. 前端「飞书项目 / 目录管理」页选空间→点「拉取/全拉」，或 `POST /api/v1/sources/feishu-project/pull`。
3. 拉取会同步飞书用户进 `users`（含 `feishu_user_key`/邮箱/头像），并把工作项 QA 解析为二级需求负责人。
4. 顶部「当前用户」选择即“我是谁”，记在浏览器 localStorage。

## 接入 AI Phone / AI Web

1. `CASE_FLOW_AIPHONE_BASE_URL` 改成部署方 AI Phone 地址。
2. 如需 Web 自动执行，`CASE_FLOW_AIWEB_BASE_URL` 改成部署方 AI Web 地址。
3. `CASE_FLOW_PUBLIC_BASE_URL` 改成当前 AI Phone 能反向访问到的 case-flow 地址（不能是 127.0.0.1）。
4. `CASE_FLOW_AIWEB_CALLBACK_BASE_URL` 改成 AI Web 能反向访问到的 case-flow 地址（不能是 127.0.0.1）。
5. 重启后端。内网部署按当前协议无需鉴权；公网或跨团队网络必须自行加网关鉴权/来源限制。

## 企业业务工具

CLI 在系统中只是一种可选 Tool。开源版本不内置公司的 CLI Tool，也不包含测试账号、权益或业务数据准备实现。需要这些能力的部署方应自行开发工具并接入 Agent 或 Hybrid；未接入时不得把数据前置假定为成功，其缺失也不影响其他已注册工具。

## 提交前自检

- 行为有变：更新 `功能说明.md` 和对应公开集成文档。
