# 后端

Case Flow 的 FastAPI 后端。

## 本地启动

```bash
/opt/homebrew/bin/python3.11 -m venv .venv
.venv/bin/python -m pip install -e ".[dev]"
cp .env.example .env
.venv/bin/alembic upgrade head
.venv/bin/uvicorn app.main:app --reload --host 127.0.0.1 --port 8800
```

## 边界

- `api` 负责把 HTTP 请求映射到领域服务。
- `services` 负责业务编排。
- `repositories` 负责持久化细节。
- `executors` 负责执行器提供方集成契约。
- `report_readers` 负责报告解析契约。
- `llm` 负责模型提示词和提供方适配器。

不要把企业专属接口或令牌放进这个仓库。

## 场景化说明

后端像工作台的大脑。前端点击“导入 Markdown”或“执行 case”时，真正的解析、碰撞判断、状态写入和执行器提交，都应该由这里的服务层和仓储层完成。
